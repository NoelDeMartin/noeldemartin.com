Last month, I finished the second part of the course and started writing an update for this journal... But I ended up scrapping it, because it was turning out very negative, and I promised I was done with that in my last entry.

Well, I just finished the course, and I'll have to publish an update because I want to be done with the task. But I don't want to be disingenuous, so beware, it's going to be negative 😅.

I don't want to repeat what I said last time, but I will emphasize a couple of points. The first one is that, indeed, the course was _too easy_. I had also been using CSS for years before I took [Josh's CSS course](https://noeldemartin.com/tasks/learning-css), but I learned a ton and many of the exercises were challenging. But that wasn't the case in this course. Even though Josh's content is really great, and I still love his teaching style, the course became a bit of a slog. Suffice to say, I'm glad to be done with it :/.

And the second point, which is probably the most relevant, is that I don't think Josh has ever worked with anything other than React. So the whole premise that this course is also right for "Angular/Vue" developers is completely flawed. I still think this course is awesome for beginners, maybe even for experienced developers who don't have a ton of frontend experience. But if you're already comfortable with any of the other modern frontend frameworks, you're better off just reading [Josh's blog](https://www.joshwcomeau.com/react/) to get "the hang of React" (I haven't found the joy, yet).

Now, I suppose I should also share my opinion about some of the new concepts I did learn. Such as React Server Components. Well, what can I say, I don't like them either 🙈. In particular, I think the way in which server code and client code is mixed up in React is awful. To make things even more confusing, React "Client" Components are also rendered on the server! And I want to clarify, I don't mean "confusing" because I don't understand how they work. Yes, Josh is awesome and everything he explains in the course is crystal clear. I just don't like the design choices made in React.

I've also been working with React a lot more since my last update, mainly on my day job (I don't use it for side-projects anymore). One of the most annoying things in React is the lack of global state. Everything has to be a damn hook, and the whole reactivity system is very counter-intuitive. I did find a library using global state, called [react-hot-toast](https://github.com/timolins/react-hot-toast), and that one is pretty cool. If you want to render a toast, you only need to import a function, and call it. Imagine that! But 99% of React libraries, require you to use hooks and deal with React's reactivity mess.

Anyhow, to end on a positive note, I'll say that I still have a module left in the course: the Layout Animations bonus module. And I'm really looking forward to that one! In fact, I've also been taking Josh's new [Whimsical Animations](https://whimsy.joshwcomeau.com/) course, and I'm enjoying it a lot.

However, none of those have anything to do with React (thankfully), so I'll consider this task done.
